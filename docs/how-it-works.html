University notes engine · build pipeline

/tailor-notes

One command turns a folder of lecture slides, worksheets and revision decks into compact, complete, uniformly formatted LaTeX notes. It reads every page as an image, crops the diagrams that matter, drafts a master markdown it can answer questions from later, and compiles a print ready PDF. This is how a run flows.

Input  PDF slides, worksheets, solutions Output  markdown + .tex + PDF Rule  each PDF vision read once, ever

Capability A

Generate a week, stage by stage

Runs autonomously from the files in a week folder to a committed PDF. Every stage writes its output to disk before the next begins, so an interrupted run resumes for free.

1

Inventory the sources

List sources/ ordered by upload time, since that is the order the material was delivered at university. Classify each file as a lecture part, worksheet, solutions, revision deck or extra.

Get-ChildItem | Sort-Object CreationTime # delivery order, not filename

→ a processing plan; generated files are never treated as source

2

Render pages to images

Each source PDF is rasterised at 150 DPI, low enough to keep vision cheap, sharp enough to read every equation and axis label.

pdftoppm -png -r 150 slides.pdf pages-cache/page

→ pages-cache/ (gitignored, regenerable)

3

Read every page, checkpointed

Pages are read by vision in small batches of five to six, and each batch is appended to an extraction file before the next is read. Text, equations to LaTeX, tables to tabular, and diagrams marked for cropping are all captured. A completed file is stamped and never read again.

extraction/slides06.md # ← appended per batch <!-- EXTRACTION COMPLETE: 31 pages -->

→ extraction/*.md · the checkpoint cache and the run's memory

4

Crop the crucial diagrams

Where a figure carries real information, a native crop is cut straight from the original page at 300 DPI. Coordinates measured on the cheap 150 DPI render are scaled up automatically, so no second tool is needed.

pdftoppm -r 300 -x -y -W -H -singlefile slides.pdf figures/fig08

→ figures/ · sharp cut outs embedded straight into the notes

5

Draft the master markdown

The extractions are synthesised into one markdown file in lecture order, compact but losing nothing. This file is the permanent recall source: tutor mode answers from it alone, never re-reading a PDF.

→ CODE_WeekNN_Notes.md

6

Transform to LaTeX and compile

The markdown becomes a .tex built on the shared style, then compiles twice from the week folder to resolve the contents page and references. Errors are read from the log and fixed until the run is clean.

$env:TEXINPUTS = ".;…/templates;" pdflatex -interaction=nonstopmode CODE_WeekNN_Notes.tex # ×2

→ CODE_WeekNN_Notes.pdf · page count and warnings reported

7

Lint, commit, push

A final pass strips any stray dash or hyphen from prose (open book exam copy safety), then the whole week is committed and pushed to the private GitHub repo.

git add -A ; git commit -m "CODE Week NN notes" ; git push

→ notes backed up and reachable on any device

Why it stays reliable

Three guardrails

The command is tuned for a Sonnet class model with thinking on. These three rules are what keep a long, high volume run accurate and resumable.

Checkpoint, never lose work

Small batches written to disk as they go mean an API error or interruption costs nothing. Rerunning skips any file already stamped complete, so no page is ever read twice.

The files are the memory

Extractions feed the markdown, the markdown feeds the PDF, and tutor mode reads only the markdown. Nothing depends on holding a 150 page week in one context.

Prescriptive formatting

Every equation gets a variable list, every worked example gets numbered steps, every box title is braced. Uniform by construction, identical every week.

Beyond generation

The other four capabilities

Same command, routed by what you ask for.

tutorAsk anything from your notes. Answers from the master markdown only, so recall costs almost nothing and never re-reads a slide.
worksheetFold in a worksheet and its solutions. Adds each solved problem to that week's worked examples, in the same step by step format, then recompiles.
new moduleScaffold a module in seconds. Creates the module record and a folder per teaching week. Works for any module, nothing is hard coded.
syncCommit and push on demand. A manual save point whenever you want one.

Your side of it

The weekly routine

One Cowork project per module, one chat per week.

  1. Drop the week's files into Week NN/sources/, uploaded in delivery order.
  2. Start a fresh chat and run /tailor-notes Week NN. It runs to a compiled PDF on its own.
  3. Ask follow up questions in the same chat all week; tutor mode recalls from the markdown.
  4. Next week, new chat. Old chats are safe to delete: everything lasting lives in the repo and on GitHub.
