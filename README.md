# uni-notes

Uniform, compact, complete university notes generated by the `/tailor-notes` Claude command. Fully generic: one repo for EVERY module, one folder per module, one folder per week, one chat per week.

## Layout

```
uni-notes/
├── .claude/commands/tailor-notes.md   # the command (auto available in any Claude/Cowork session on this repo)
├── templates/
│   ├── uninotes.sty                   # ALL shared styling: change here, restyle every week of every module
│   ├── week_template.tex              # skeleton each week's .tex is built from
│   └── module_template.md             # blank module.md, copied when scaffolding a module
├── scripts/render_pages.ps1           # PDF -> page PNGs (150 DPI) and figure crops (300 DPI)
└── <MODULE CODE>/                     # one folder per module, e.g. ES3E8/, ES327/, ES3C5/
    ├── module.md                      # module code + full name (drives every header)
    └── Week NN/
        ├── sources/                   # upload lecture PDFs, worksheets, solutions here
        ├── extraction/                # generated: per file vision extraction (checkpoint cache)
        ├── figures/                   # generated: cropped diagrams from the original slides
        ├── pages-cache/               # generated, gitignored: rendered page images
        └── <CODE>_WeekNN_Notes.md/.tex/.pdf   # generated outputs
```

## Adding a module (once per module, seconds)

In any chat: `/tailor-notes set up module ES327 Design of Mechanical Systems` (capability D). It creates `ES327/module.md` from the template plus `Week 01..NN/sources/` folders. Or copy `templates/module_template.md` to `<CODE>/module.md` by hand and fill it in.

## Weekly routine (Cowork)

1. Open the module's Cowork project (see Cowork setup below).
2. Drop the week's files into `Week NN/sources/` **in delivery order** (the pipeline processes files by creation time, so upload Part 1 before Part 2, revision deck last).
3. Start a **new chat** and run: `/tailor-notes ES327 Week 5` (module code optional when the project folder is the module folder).
4. Ask follow up questions in the same chat; tutor mode answers from the generated markdown without re-reading any PDFs.
5. Next week: new chat. Old chats can be deleted safely; everything permanent lives in this repo, backed up on GitHub.

## Cowork setup (once per module)

1. Cowork > New project, named after the module (e.g. "ES327 Notes").
2. Set the project's folder to `C:\Users\mryx1\repos\uni-notes\<MODULE CODE>` (or the repo root if you prefer one project for everything).
3. The command is available in every chat automatically: globally from `C:\Users\mryx1\.claude\commands\tailor-notes.md`, and from this repo's `.claude/commands/` when the project folder is the repo root.

## GitHub (one time)

```powershell
gh auth login          # browser login, choose GitHub.com + HTTPS
gh repo create uni-notes --private --source "C:\Users\mryx1\repos\uni-notes" --push
```

After that the command commits and pushes automatically at the end of every generation run. Manual sync any time: `/tailor-notes sync`.

## Compile a week by hand (if ever needed)

```powershell
Set-Location "C:\Users\mryx1\repos\uni-notes\<MODULE>\Week NN"
$env:TEXINPUTS = ".;C:\Users\mryx1\repos\uni-notes\templates;"
pdflatex -interaction=nonstopmode <CODE>_WeekNN_Notes.tex   # run twice
```

## Formatting contract (identical for every module, enforced by the command)

- Header: `MODULECODE: Module Name - Week N Notes`; page numbers centred in the footer; one page contents; tight booklet margins.
- No dashes or hyphens in prose (open book copy safety); hyphenation at line breaks disabled.
- Every equation centred with a `Term: FullName (UNITS)` variable list on first use.
- Worked examples and revision questions in Q&A format with `Step X:` explanations mirroring the official solutions.
- Key Summary section at the end of every week.
